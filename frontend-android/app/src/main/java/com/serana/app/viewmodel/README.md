# viewmodel 目录说明

这里放页面状态和与后端交互的编排逻辑。

## 当前重点

- `ChatViewModel.kt`
  - 聊天流式消息、审批事件、会话刷新
- `SkillsViewModel.kt`
  - 技能列表刷新
  - 远程技能安装审批
  - 远程技能更新审批
  - 技能生命周期状态加载
  - 技能生效范围切换
  - 本地技能卸载审批
  - 本地 ZIP 技能导入与审批续传

## 维护约定

- 页面展示用的短期 UI 状态放这里
- 审批两步流在这里闭环：先提交审批，再继续原动作
- 如果后端新增技能动作状态，优先从这里补状态机，再改页面展示
- 流式聊天的 `done` 事件会先合并最终 `tool_calls` 和 `thinking_blocks`，debug 补水只负责校准落库后的最终消息。
